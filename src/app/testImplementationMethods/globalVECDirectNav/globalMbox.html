Used with VEC: Yes

Implementation

This is the same as the other global mbox but with VEC setup using direct-nav

Rationale:

You have a straightforward "replace content" style split test and the Adobe Visual Experience Composer readily handles the types of content replacement you are doing. Simply the inclusion of at.js should support this.

The Test may not have worked. This is default content

Pros:

  • Easy setup
  • For simple tests this is very marketing-friendly
  • Potentially not tightly coupled with code
Cons:
  • If customization is required (e.g. page params) then code changes are needed
  • Customizations such as above must be positionally aware - i.e. come before at.js
  • "Route monitoring" for detecting page state to apply test may have caveats (TBD)
