Rationale:

You have an "offer" - an Adobe concept of "html snippet" defined in the Content section of Adobe Target. You write specific code to pull down that offer and the Adobe code replaces the content by locating an mbox and inserting content.

The Test may not have worked. This is default content

Pros:

  • This is really just offered as a migration path
Cons:
  • Not intended to be a forward-path
  • Not viable with VEC as the offer to be presented is "inline" in the code
